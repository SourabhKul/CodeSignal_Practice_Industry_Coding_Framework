# Scenario Catalog

## Progressive OA Families

Practice these first:

1. `in_memory_db` - CRUD, scans, TTL, backup/restore.
2. `banking_system` - balances, ranking, pending transfers, merge/history.
3. `cloud_storage` - file search, ownership/capacity, backup/restore.
4. `recipe_manager` - CRUD, sorting, permissions, version history.
5. `employee_system` - attendance, ranking, promotions, pay windows.
6. `task_manager` - priority search, quotas, TTL, overdue history.
7. `course_registration` - registration, pairing, GPA, department ranking.

Exact APIs rotate. The goal is to become fast at state modeling, deterministic sorting, half-open time intervals, historical snapshots, and backward-compatible refactoring rather than memorizing a single answer.
